# Requerimientos - Asgard Pets v2.0

## Descripción del proyecto

Asgard Pets es una distribuidora de alimento, medicina y accesorios para mascotas, los dueños de Asgard crearon "Aurora" una fundación sin ánimo de lucro que rescata mascotas para darlas en adopción. Asgard usa parte de sus ganancias para el sostenimiento de Aurora. 

Asgard Pets y Aurora buscan el cuidado y bienestar de las mascotas, realizando una labor social que incentiva a los ciudadanos a proteger los animales.

Este sistema busca tener un manejo del inventario y también de los usuarios que van a ser parte de Asgard Pets.

## Problema que resuelve

El inventario actual tiene opciones limitadas, además que no cuenta con la administración de usuarios que pertenecen a Asgard.

## Objetivo general

El objetivo de esta versión será implementar un inventario más versátil y administrar los usuarios.

## Objetivos de aprendizaje

- Aplicar los principios de POO.
- Comprender el uso de interfaces, sobrescritura y sobrecarga de métodos.
- Diseñar clases aplicando buenas prácticas de programación.
- Practicar el uso de git y github como si fuera un entorno real.

## Usuario del sistema

El usuario será un empleado de Asgard Pets autorizado para manipular los diferentes modulos del sistema. 
## Alcance del sistema

- Gestión de Inventario.
-  Gestión de Usuarios.
- Gestión de Ventas.
- Gestión de la Fundación Aurora.

## Requisitos funcionales

RF - 01
El sistema debe mostrar un menú general que permita al empleado acceder a los diferentes módulos del sistema.
RF - 02
El sistema permitirá gestionar el inventario, permitiendo registrar, buscar, actualizar, eliminar un ítem o mostrar todo el inventario.

RF - 03
El sistema permitirá registrar, mostrar y buscar usuarios nuevos.

RF -04
El sistema debe permitir gestionar las ventas, permitiendo registrar, consultar, cancelar o mostrar la factura.

RF - 05 
El sistema mostrará información sobre la fundación Aurora y permitirá registrar mascotas rescatadas, adopciones y donaciones.

RF - 06
El sistema deberá permitir al usuario navegar entre los diferentes módulos y regresar al menú principal.

## Requisitos no funcionales

RNF - 01
El sistema será desarrollado en Java.

RNF - 02
El sistema funcionará mediante una interfaz de consola.

RNF - 03
La información no será persistente; los datos se perderán al cerrar la aplicación.

RNF - 04
El proyecto se desarrollará aplicando Programación Orientada a Objetos.

RNF - 05
El proyecto usará herramientas como Git, Github e IntelliJ.

## Fuera de alcance

- EL proyecto no será una app web ni móvil.
- No tendrá base de datos.
- No tendrá persistencia.
- No habrá inicio de sesión.
- No se realizarán pagos en línea.
